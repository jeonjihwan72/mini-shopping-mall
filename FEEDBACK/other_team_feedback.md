- 생성형 AI에서 `Bold`체를 사용하면 생성형 AI는 이를 최중요 사항으로 인식한다는 점을 주의할 것
- 기대효과와 같이 사실상 의미없는 문구 제거할 것 (생성형 AI가 사용하는 토큰이 낭비되는 것을 줄일 수 있음)
- 데이터베이스 스키마 작성할 것
- 현재 진행하는 프로젝트가 디자인 패턴 적용이 핵심이라는 점은 간과하지 말고 범위를 규정할 것
- firebase를 활용한 알림시스템은 firebase 자체의 기능이 있기 때문에 시스템 개발이 아닌 운영 (DevOps) 분야로 생성형 AI가 변질될 위험이 있음
- firebase를 사용하는 것은 firebase 자체에서 지원하는 기능이 많기 때문에 사용하는 것을 비추천
- UI 요소를 최소화하는 것을 추천: 수업의 목적과는 거리가 멀기 때문
- `aceptance.md` 파일은 사람이 직접 테스트 케이스를 실행해보는 것이 아니라 AI에게 테스트 검수 과정을 맡길 수 있도록 보충할 것
- 생성형 AI에 반복해서 입력 출력과정을 거쳐 사전 문서를 최대한 구체화하는 것이 중요
- 실질적인 코드 생성은 `instruction.md`, `specification.md`를 생성형 AI에 넣어 나온 출력물을 기준으로 한다.
