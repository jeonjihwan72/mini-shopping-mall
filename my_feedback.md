- 프론트엔드 (UI) 부분은 필수 기능을 제외하고는 모두 제거할 것 : 수업의 목표와 무관하며 코드 라인 비중만 차지함
- UI 부분은 필수 상호 작용 요소만 넣고 내부 동작의 경우 콘솔로 보여줘도 무관하다.
- `acceptance.md` 파일은 사람의 기준이 아니기에 상당히 구체화 시킬 것
  - ex. 가 - 1 product.json을 출력한다 -> DB의 내용과 출력된 product.json 파일의 스키마와 내용이 일치해야 한다.
- `introduction.md` 에서 기능을 제시할 때 순차적으로 나누더라도 클래스로 묶이지 않을 확률이 높다.
- `introduction.md` 에서 시나리오 형식으로 제시하기보다는 큰 틀을 우선적으로 잡아놓고 기능 구체화 단계에서만 사용하는 것을 권장
